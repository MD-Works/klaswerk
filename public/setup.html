✦   Getting Started

Welcome to the Setup Wizard

This wizard will walk you through every step to deploy a new KlasWerk site for a client — from scratch, even if you've never done it before.

📋 What you'll need before you start
  1. A Supabase account (free) — supabase.com
  2. A PayFast account — payfast.co.za (only needed for paid courses)
  3. A Whereby account — whereby.com/embedded (only needed for live sessions)
  4. A Resend account (free) — resend.com (for email notifications)
  5. A Cloudflare account (free) — cloudflare.com (for the payment webhook and hosting)
  6. Node.js installed on your computer — nodejs.org (choose the LTS version)
  7. The KlasWerk codebase — either cloned from GitHub or the scaffold zip
🗂 What this wizard does

At the end of the 6 steps, you'll have:

  1. A complete config.js file to drop into the project
  2. All the commands you need to run, ready to copy-paste
  3. A client record saved to your MD Works registry for future maintenance
💡 How to use this page

Work through each step in order. Every step has a help section explaining what to do and why. Commands are shown for both Windows (PowerShell) and Mac/Linux (Terminal). You can come back to any step using the numbered dots above.

✦   Step 2 of 6

Supabase — Database & Auth

Supabase is the database that powers KlasWerk. It stores all users, courses, lessons, and payments — completely free on the Starter plan.

🚀 How to create a new Supabase project
  1. Go to supabase.com and sign in (or create a free account)
  2. Click New Project
  3. Choose your organisation (or create one)
  4. Give the project a name — e.g. klaswerk-clientname
  5. Set a strong database password — save this somewhere safe, you'll need it later
  6. Choose region: South Africa (Cape Town) — closest to your users
  7. Click Create new project and wait about 60 seconds
🔑 Where to find your keys
  1. In your Supabase project, go to Settings → API (left sidebar)
  2. Under Project URL — copy the URL (looks like https://xxxx.supabase.co)
  3. Under Project API keys — copy the anon / public key (starts with eyJ)
  4. Do NOT use the service_role key here — that is only for the Cloudflare Worker (configured later)
🗄 Running the database migrations

KlasWerk needs 4 SQL files run in order to set up all the tables, policies, and views. Do this once after creating the project:

  1. In Supabase, click SQL Editor in the left sidebar
  2. Click New query
  3. Open the file supabase/migrations/001_initial.sql from the project folder, copy all the text, paste it in, and click Run
  4. Repeat for 002_s8_analytics_views.sql, then 003_s9_cert_storage_public_page.sql, then 004_s10_trainer_profile_view.sql
  5. Each one should say "Success. No rows returned" — that's correct

If you see an error, check that you ran them in order (001 first).

📧 Setting up Supabase Auth email templates
  1. In Supabase, go to Authentication → Email Templates
  2. For Confirm signup: paste the contents of supabase/email-templates/confirm-signup.html
  3. For Reset password: paste supabase/email-templates/reset-password.html
  4. For Magic link: paste supabase/email-templates/magic-link.html
  5. Click Save after each one

PayFast — Payments

PayFast handles all payments securely. You need this for paid courses. If this client only runs free courses, you can skip this step.

🏦 Setting up your PayFast account
  1. Go to payfast.co.za and register as a merchant
  2. Complete the verification process (ID, bank account) — this can take 1–2 business days
  3. Once approved, log in to your merchant dashboard
  4. Go to Settings → Integration
  5. Find your Merchant ID and Merchant Key
  6. Set a Passphrase — this is a secret you choose and type into PayFast's settings. Use something strong like a random 16-character string. Write it down.
🔧 Deploying the PayFast Cloudflare Worker

This worker receives payment notifications from PayFast and updates your database. Run these commands in order:

# Navigate to the worker folder
cd workers\payment-webhook

# Install Wrangler (Cloudflare's deploy tool) — only needed once
npm install -g wrangler

# Log in to Cloudflare (opens browser)
wrangler login

# Create the KV namespace for replay protection
wrangler kv:namespace create SEEN_PAYMENTS
# Copy the "id" value from the output
# Open wrangler.toml and paste it next to: id = "REPLACE_WITH..."

# Deploy the worker
wrangler deploy

# Set secrets (you'll be prompted to type each value)
wrangler secret put SUPABASE_URL
wrangler secret put SUPABASE_SERVICE_KEY
wrangler secret put PAYFAST_PASSPHRASE
wrangler secret put PAYFAST_MERCHANT_ID
wrangler secret put TEST_MODE
# For TEST_MODE: type  false  for production, or  true  for testing

✦   Step 4 of 6

Whereby + Brand

Whereby powers the live video sessions. Brand settings let you customise the site name and logo per client.

📹 Setting up Whereby Embedded
  1. Go to whereby.com/embedded and sign up or log in
  2. Create an organisation (or use an existing one)
  3. Go to Configure → API
  4. Click Generate API key — copy it immediately, it won't show again
  5. Whereby charges per room-minute used. On the free plan you get a small monthly allowance — enough for testing.

If this client won't use live sessions, leave the API key blank — the live session features will simply not appear.

📧 Setting up Resend (email notifications)
  1. Go to resend.com and create a free account
  2. Add and verify your sending domain (e.g. klaswerk.co.za) under Domains
  3. Go to API Keys and create a new key
  4. Then run these commands to set up the Edge Functions:
# Install Supabase CLI — only once
npm install -g supabase

# Log in to Supabase CLI
supabase login

# Link to your project (get the project ref from Supabase Settings → General)
supabase link --project-ref YOUR_PROJECT_REF

# Set email secrets
supabase secrets set RESEND_API_KEY=re_xxxxxxxxxxxx
supabase secrets set APP_URL=https://yoursite.co.za
supabase secrets set FROM_EMAIL=noreply@yoursite.co.za

# Deploy edge functions
supabase functions deploy send-certificate-email
supabase functions deploy send-session-reminder
supabase functions deploy send-payment-receipt

• certificates INSERT → send-certificate-email

Save Client Record

Save this client's details to your MD Works registry for future maintenance. You'll be able to look up any client's Supabase project, domain, and contact at any time.

🗄 First time? Set up your MD Works Registry

The registry uses your own personal Supabase project — separate from any client project. You only do this once.

  1. Create a new Supabase project called mdworks-registry at supabase.com
  2. In the SQL Editor, run this to create the clients table:
CREATE TABLE public.kw_clients (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  created_at      TIMESTAMPTZ DEFAULT NOW(),
  client_name     TEXT NOT NULL,
  trading_name    TEXT,
  contact_name    TEXT,
  contact_email   TEXT,
  contact_phone   TEXT,
  domain          TEXT,
  supabase_url    TEXT,
  deploy_date     DATE DEFAULT CURRENT_DATE,
  payfast_live    BOOLEAN DEFAULT FALSE,
  whereby_active  BOOLEAN DEFAULT FALSE,
  notes           TEXT,
  features        JSONB DEFAULT '{}'
);

ALTER TABLE public.kw_clients ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Service role full access"
  ON public.kw_clients
  USING (TRUE)
  WITH CHECK (TRUE);
